#<a name="how-to-get-appsource-certified-for-azure-active-directory-ad"></a>Jak uzyskać certyfikat AppSource dla Azure Active Directory (AD) 

Aby otrzymywać certyfikacji AppSource dla Azure AD, aplikacji zaimplementować logowania wielu dzierżawy we wzorcu z Azure AD przy użyciu protokołów łączenie OpenID, OAuth 2.0 lub SAML 2.0. 

Jeśli nie znasz logowania Azure AD lub opracowywania aplikacji wielu dzierżawy:

1. Zacznij od informacji o [przeglądarce scenariuszy aplikacji sieci Web w scenariuszach uwierzytelniania dla Azure AD][AAD-Auth-Scenarios-Browser-To-WebApp].  
2. Następnie zapoznaj się z Azure AD [przewodniki szybki aplikacji sieci web start][AAD-QuickStart-Web-Apps], której pokazano, jak zaimplementować logowania i zawiera przykłady kodu pomocniczym. 

    > [AZURE.TIP] Wypróbuj Podgląd programu nasz nowy [dzięki portalowi deweloperów](https://identity.microsoft.com/Docs/Web) , która ułatwi rozpoczęcie pracy z usługą Azure Active Directory w ciągu kilku minut!  Dzięki portalowi deweloperów przeprowadzi Cię przez proces rejestracji aplikacji i integracji Azure AD do kodu.  Po zakończeniu, konieczne będzie prostej aplikacji uwierzytelnieni użytkownicy w Twojej dzierżawy i wewnętrznej, które można zaakceptować tokeny sprawdzana poprawność.

3. Aby dowiedzieć się, jak wdrażać wielu dzierżawy logowania deseniu z usługą Azure Active Directory, zapoznaj się z [jak logować się każdy użytkownik Azure Active Directory (AD) za pomocą deseń wielu dzierżawy aplikacji][AAD-Howto-Multitenant-Overview]
